# ArtWallet -- States & Transitions

## State Machine Diagram

```
┌─────────┐
│  INIT   │── есть vault ──────────────> UNLOCK
└────┬────┘
     └── нет vault ────────────────────> ONBOARDING


┌──────────────┐
│  ONBOARDING  │── "Create new" ───────> CREATE
└──────┬───────┘
       └── "Import existing" ──────────> IMPORT


┌──────────┐
│  CREATE  │── подтвердил мнемонику ───> SET_PASSWORD
└──────────┘


┌──────────┐
│  IMPORT  │── валидная мнемоника ─────> SET_PASSWORD
└────┬─────┘
     └── невалидная ───────────────────> (ошибка, остаемся)


┌──────────────┐
│ SET_PASSWORD │── пароль задан ───────> DASHBOARD
└──────────────┘


┌──────────┐
│  UNLOCK  │── верный пароль ──────────> DASHBOARD
└────┬─────┘
     ├── неверный пароль ──────────────> (ошибка, остаемся)
     └── "Reset wallet" ──────────────> ONBOARDING


┌─────────────┐
│  DASHBOARD  │── "Receive" ──────────> RECEIVE
└──────┬──────┘
       ├── "Send" ────────────────────> SEND
       ├── "Settings" ────────────────> SETTINGS
       └── auto-lock timeout ─────────> UNLOCK


┌───────────┐
│  RECEIVE  │── "Back" ───────────────> DASHBOARD
└───────────┘


┌────────┐
│  SEND  │── валидация ошибка ────────> (показать ошибку, остаемся)
└───┬────┘
    ├── address guard сработал ───────> SEND_WARNING
    └── всё ок ──────────────────────> SEND_CONFIRM


┌──────────────┐
│ SEND_WARNING │── "Send anyway" ─────> SEND_CONFIRM
└──────┬───────┘
       └── "Cancel" ─────────────────> SEND


┌──────────────┐
│ SEND_CONFIRM │── "Confirm" ─────────> SEND_PENDING
└──────┬───────┘
       └── "Cancel" ─────────────────> SEND


┌──────────────┐
│ SEND_PENDING │── успех ─────────────> SEND_SUCCESS
└──────┬───────┘
       └── ошибка ───────────────────> SEND_ERROR


┌──────────────┐
│ SEND_SUCCESS │── "Done" ────────────> DASHBOARD
└──────────────┘


┌────────────┐
│ SEND_ERROR │── "Retry" ─────────────> SEND_CONFIRM
└─────┬──────┘
      └── "Cancel" ──────────────────> DASHBOARD


┌────────────┐
│  SETTINGS  │── "Back" ─────────────> DASHBOARD
└─────┬──────┘
      └── "Reset wallet" ───────────> ONBOARDING
```

## States Summary

| # | State | Description |
|---|-------|-------------|
| 1 | `INIT` | Проверка наличия vault в storage |
| 2 | `ONBOARDING` | Выбор: создать или импортировать кошелек |
| 3 | `CREATE` | Показ мнемоники + подтверждение (выбрать слова) |
| 4 | `IMPORT` | Ввод 24 слов мнемоники |
| 5 | `SET_PASSWORD` | Установка пароля для шифрования vault |
| 6 | `UNLOCK` | Ввод пароля для расшифровки vault |
| 7 | `DASHBOARD` | Главный экран: адрес, баланс, история транзакций, поиск |
| 8 | `RECEIVE` | Адрес кошелька + кнопка копирования (+ QR бонус) |
| 9 | `SEND` | Форма: адрес получателя, сумма, валидация |
| 10 | `SEND_WARNING` | Предупреждение address guard |
| 11 | `SEND_CONFIRM` | Финальное подтверждение: адрес + сумма + fee |
| 12 | `SEND_PENDING` | Ожидание ответа сети (спиннер) |
| 13 | `SEND_SUCCESS` | Tx отправлена + ссылка на explorer |
| 14 | `SEND_ERROR` | Описание ошибки + retry/cancel |
| 15 | `SETTINGS` | Auto-lock timeout, reset wallet |

## Address Guard Triggers

Предупреждение (`SEND_WARNING`) показывается когда:

| Сценарий | Описание |
|----------|----------|
| Новый адрес | Адрес не из whitelist (первая отправка на этот адрес) |
| Clipboard hijack | Адрес изменился между моментом paste и нажатием Send |
| Self-send | Адрес совпадает или похож на собственный адрес кошелька |
| Address poisoning | Адрес похож на известный из whitelist, но отличается в середине |

## Validation Rules (SEND)

| Проверка | Условие ошибки |
|----------|---------------|
| Адрес пустой | Поле не заполнено |
| Адрес невалиден | Не соответствует формату TON-адреса |
| Сумма пустая | Поле не заполнено |
| Сумма <= 0 | Отрицательная или нулевая сумма |
| Сумма > баланс | Недостаточно средств (с учетом fee) |
| Сумма не число | Невалидный ввод |

## Auto-Lock

Переход `* -> UNLOCK` происходит при:
- Неактивность N секунд (настраивается, default: 60с)
- Вкладка скрыта (`visibilitychange`)
- Закрытие popup расширения (контекст уничтожается автоматически)
